# 破产边缘 (Edge of Default)

## 一句话描述

一款以主权债务危机为背景的多模式金融博弈策略游戏，玩家扮演政府决策者（拯救者）或市场投机者，通过政策制定或市场操作从国家破产危机中脱困或获利。

---

## 游戏类型与定位

- **类型**：金融博弈策略游戏 / 经济危机模拟
- **目标用户**：对宏观经济、金融市场、地缘政治感兴趣的用户；喜欢策略决策、数据分析的玩家
- **核心体验**：不对称信息博弈、危机决策的紧迫感、从危机中寻找盈利机会的刺激感
- **视觉风格**：轻量级 2D 界面，数据仪表盘风格，危机感用红色调渲染，非写实卡通风格

---

## 游戏模式

### 🟢 拯救者模式（Savior）

玩家扮演政府决策者，对抗投机者 AI。

**目标**：在 30 回合内，通过政策工具组合稳定经济，避免国家破产（信用评级归零）。

**核心机制**：
- **回合结构**：`AI 行动 → 玩家决策 → 回合结算 → 结果反馈`
- **四大政策工具**：货币政策（利率/汇率）、财政政策（税收/支出）、资本管制、国际求援
- **AI 投机者行为**：做空汇率、做空国债、做空股指、做空本币债务；在特定条件触发时发动协同攻击
- **关键指标**：信用评级（核心胜负线）、通胀、外汇储备、汇率、公众支持率
- **危机升级路径**：单一指标恶化会传导至其他指标，形成系统性螺旋

**特色**：
- 拯救者模式的 AI 投机者有明确的攻击逻辑（会在你脆弱时联合出击）
- 存在"剧情事件"系统——重大危机事件（如银行挤兑、外债违约）以叙事卡片形式出现，玩家选择应对方式

---

### 🔴 投机者模式（Speculator）

玩家扮演市场投机者，从危机中获利。

**目标**：在 30 回合内，将初始资金扩大 3 倍（胜利条件）或亏光（失败条件）。

**核心机制**：
- **回合结构**：玩家自主选择操作顺序 → 点击"结束本月"推进回合 → 系统执行市场波动计算
- **四大操作维度**：
  1. **情报购买**：信息不对称是核心武器；免费情报粗糙，付费情报精准；付费情报下回合过期但会保留显示
  2. **交易终端**：做多/做空汇率、国债、股指；实时盯盘，设置止损止盈
  3. **操控行动**（高级）：直接干预市场指标，有 cooldown 冷却机制（3~4回合），成功率有限，带副作用
  4. **剧情事件**：与拯救者模式共享剧情事件池，触发后影响市场走向
- **关键指标**：现金、持仓盈亏、总资产、回合数
- **政府 AI 日志**：告诉玩家政府正在做什么，影响你的交易决策

**特色**：
- 信息不对称是唯一武器——好的情报意味着提前布局
- 操作顺序没有强制约束——玩家自行决定先交易还是先读情报
- 操控行动有 cooldown——不能无脑连续使用，需等待恢复

---

### ⚔️ 对抗模式（Versus）

双人同屏博弈：一人扮演拯救者，一人扮演投机者，在同一回合结构中直接对抗。目前处于早期实现阶段。

## 核心数据模型

### 市场状态（MarketState）

| 字段 | 说明 | 初始值（投机者）| 正常范围 |
|------|------|----------------|----------|
| `exchange_rate` | 本币汇率（兑美元）| 0.92 | 0.8 ~ 1.2 |
| `inflation` | 通胀率（%） | 45 | < 10% |
| `credit_rating` | 主权信用评级（0~100）| 38 | > 60 |
| `bond_price` | 国债价格 | 0.18 | 0.5 ~ 1.0 |
| `stock_index` | 股指 | 3200 | 3000 ~ 5000 |

### 资产结构（SpeculatorAssets）

- `cash`：可用资金
- `positions`：持仓列表，每条包含品种、方向（做多/做空）、数量、盈亏
- `total_value`：总资产 = 现金 + 持仓市值
- `pnl_today`：当日盈亏

### 持仓（Position）

```typescript
interface Position {
  id: string;
  type: 'exchange_rate' | 'bond' | 'stock';
  direction: 'long' | 'short';
  amount: number;        // 投入金额
  entry_price: number;   // 入场价格
  current_price: number; // 当前价格（每回合更新）
  pnl: number;           // 绝对盈亏金额
}
```

### 操控行动（ManipulationAction）

```typescript
interface ManipulationAction {
  id: string;
  name: string;
  description: string;
  cost: number;           // 操作成本
  effects: Partial<MarketState>;  // 对市场指标的影响
  side_effect: string;   // 副作用描述
  success_rate: number;   // 成功率（0~1）
  cooldown: number;       // 冷却回合数
  is_cooling: boolean;   // 当前是否在冷却
  cooldown_left: number;  // 剩余冷却回合
}
```

## 回合结算机制（投机者模式）

每次点击"结束本月"后，系统按以下顺序执行：

1. **剧情检测**：根据当前市场状态判断是否触发重大事件（剧情系统）
2. **持仓结算**：所有持仓按当前市场价格重估，触发止盈/止损则强制平仓，产出通知
3. **市场波动**：基于随机噪声 + 政府行动 + 通胀惯性 + 操控行动残余影响，计算下回合各指标
4. **操控冷却倒计时**：所有正在冷却的操控行动 -1，到 0 时解除冷却
5. **操控刷新**：每3回合随机替换1个非冷却中的操控行动
6. **情报过期**：旧情报标记为已过期并保留显示（保留最多6条），同时生成新情报池
7. **政府行动日志**：记录 AI 政府本回合做了什么（文本描述）
8. **胜负检测**：检测总资产是否触及胜利（×3）或失败（≤0）条件

---

## 剧情系统

与拯救者模式共享事件池。重大危机以叙事卡片形式出现，带多个选项，每个选项有成功率（影响市场指标）和代价描述。

**事件类型示例**：
- 银行挤兑（选项：救市 / 任其倒闭）
- 外债违约（选项：申请援助 / 强硬谈判）
- 资本外逃（选项：实施管制 / 提高利率）

## 当前已知问题与优化方向

### 待优化项（P1）

1. **初始市场过于危险**：游戏一开始各指标就处于危机区间（通胀45，信用38），玩家难以找到入场节奏，建议前5回合减少偏差力度
2. **操控冷却提示不完整**：触发成功后通知只说"成功"，没说"进入X回合冷却"
3. **通知消失太快**：5000ms 自动消失，建议延长到 8000ms
4. **政府行动日志是静态文本**：目前是固定文案池，应该改为真正联动市场状态的动态描述
5. **缺乏操作引导**：新手不知道应该先买情报还是先交易

### 体验增强方向（P2）

1. **H5 端交易终端操作体验**：当前 BottomSheet 操作流程不够顺畅
2. **胜/负结算界面**：目前较为简陋，缺乏复盘数据展示
3. **存档/读档**：目前每次刷新页面游戏重置
4. **音效与震动反馈**：关键操作（开仓、平仓、触发操控）缺少触觉反馈
5. **成就系统**：积累一定交易次数、达成特定市场条件后解锁称号

## 游戏核心哲学

> **信息不对称是唯一的武器，危机是唯一的舞台。**

- 拯救者的乐趣在于**在不可能中寻找最优解**——资源有限，信息有限，时间有限
- 投机者的乐趣在于**预判与执行**——读懂情报，快人一步，在混沌中盈利
- 双方都在与**系统性风险**博弈——没有完美的策略，只有适时的决策
